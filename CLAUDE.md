## Project Overview

Montenegro Railways Timetable Telegram bot (`@Monterails_bot`). Go 1.23 project that runs as a stateless webhook-based HTTP server on Google Cloud Run. The timetable is pre-compiled into Go code (no database).

## Common Commands

```bash
# Testing
make test_unit           # Unit tests only (go test -count=1 -race ./internal/...)
make test_integration    # Integration tests (requires tdlib installed)
make test                # All tests

# Run a single test
go test -count=1 -race -run TestName ./internal/path/to/package/...

# Run server locally
go run ./cmd/tg-server

# Regenerate timetable from zpcg.me
make parse_timetable     # runs: go run ./cmd/exporter

# Build & deploy
make build               # Docker build
make new_version         # Full pipeline: parse + build + push + deploy
```

## Architecture

**Entry points** (`cmd/`):
- `tg-server` — Main HTTP server handling Telegram webhooks
- `exporter` — Parses zpcg.me API, generates Go timetable code into `gen/timetable/timetable.gen.go`
- `tg-init` — One-time bot setup (commands, description)

**Request flow:**
Telegram webhook POST → `server.Handler` → `app.HandleUpdate()` → resolve station names → find routes (pathfinder) → render response → send back via gotgbot

**Key packages** (`internal/`):
- `app/` — Orchestrator: dispatches messages/callbacks to services
- `server/` — HTTP handler, Telegram bot integration, middleware
- `config/` — Env-var config loading (`caarlos0/env`)
- `service/pathfinder/` — Route-finding: direct paths, then transfer via Podgorica
- `service/parser/` — HTML/JSON parsing of zpcg.me
- `service/name/` — Station name resolution (exact match → fuzzy match, Latin+Cyrillic)
- `service/render/` — Multi-language response rendering (9+ languages)
- `service/callback/` — Inline button callback parsing (update date, reverse route)
- `service/blacklist/` — Filters out non-functional stations
- `model/timetable/` — Core data structures (Station, Train, Path)
- `model/render/` — Per-language message templates in `{lang}/` subdirs
- `model/stations/` — Station alias definitions and blacklist

**Generated code** (`gen/`):
- `timetable/timetable.gen.go` — Auto-generated compiled timetable (~4000 lines). Do not edit manually.
- `mocks/` — Test mocks (generated by mockery, config in `.mockery.yaml`)

## Key Design Decisions

- **Stateless & in-memory only**: No database, no session storage. All timetable data is compiled into the binary.
- **Webhook-based**: No polling; Google Cloud Run scales to zero when idle.
- **Pathfinding**: Direct routes checked first. If none found, tries one-transfer routes via Podgorica.
- **Station name matching**: Normalizes to Latin, removes diacritics, then tries exact match before approximate/fuzzy.
- **Language detection**: Uses Telegram's `language_code` field with English fallback.
